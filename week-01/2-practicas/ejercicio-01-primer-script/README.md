# Ejercicio 01: Mi Primer Script

## 🎯 Objetivo

Escribir y ejecutar tu primer programa JavaScript usando `console.log()` para mostrar diferentes tipos de datos en la consola.

## ⏱️ Duración estimada

35 minutos

## 📋 Instrucciones

Abre `starter/index.js` y sigue los pasos descomentando el código de cada sección. Ejecuta el archivo con `node starter/index.js` después de cada paso para ver el resultado.

---

### Paso 1: Hola, mundo

El primer programa en cualquier lenguaje — mostrar un mensaje en pantalla.

```javascript
// Así se muestra texto en la consola
console.log("¡Hola, mundo!");
```

**Abre `starter/index.js`** y descomenta la sección del Paso 1.

---

### Paso 2: Mostrar texto

Un string puede ser cualquier texto entre comillas.

```javascript
// Los strings van entre comillas simples o dobles
console.log("Estoy aprendiendo JavaScript");
console.log("También funciona con comillas dobles");
```

**Descomenta la sección del Paso 2.**

---

### Paso 3: Mostrar números

Los números no necesitan comillas. Pueden ser enteros o decimales.

```javascript
// Número entero — sin comillas
console.log(42);

// Número decimal — el separador es el punto
console.log(3.14);

// Número negativo
console.log(-10);
```

**Descomenta la sección del Paso 3.**

---

### Paso 4: Mostrar booleans

Solo hay dos valores posibles: `true` o `false`.

```javascript
// Boolean — sin comillas
console.log(true);
console.log(false);
```

**Descomenta la sección del Paso 4.**

---

### Paso 5: Organizar la salida con separadores

Cuando tienes mucha salida en consola, los separadores la hacen legible.

```javascript
// Usar console.log() con texto vacío crea una línea en blanco
console.log("");

// Usar console.log() con guiones crea un separador visual
console.log("----------------------------");
```

**Descomenta la sección del Paso 5.**

---

## 🏃 Cómo Ejecutar

```bash
node starter/index.js
```

## ✅ Criterios de Éxito

- [ ] El script corre sin errores con `node`
- [ ] La consola muestra texto, un número y un boolean
- [ ] La salida está organizada y es legible
